Processes Interface:

 

Algorithm Process:

One of ATAT’s main purposes is algorithm development. ATAT allows for many algorithms to be tested within one experiment. The way this is implemented in through the processes interface. There are a few keywords to discuss. Processes will be defined as a function that results in a new image. Algorithms will be defined as a series of processes. Algorithms will use stages to keep track of the process being implemented. Each test will start at stage one. Every process in stage one will be implemented and create new images. The new, stage one, images will be used in stage two processes. Then the stage two results will be used in the stage three process, and so on until each unique algorithm is completed. This will result in a final image count as the product of the number of processes in each stage.

Once processes are added, the total number of algorithms per image are displayed. To determine the total number of result image from this, multiply the “Total algorithms” count by the total images. This number will also be displayed on the “Test and Compare” Interface.

 

Adding a Process:

1.      To select a process, click the dropdown menu under “Select Process”. This will result in a list of all the available function within ATAT. To choose one, simply click it. One process can be selected at a time.

2.      Choose the stage number. The first stage must be 1, with each continuing stage incremented by 1. There can be multiple processes per stage.

3.      Determine Parameters. Each time a process is chosen from the list, the parameters descriptions and ranges will change. Greyed out parameters will not be used. The left of the numbers box for each parameter will be a brief terminology description of the parameter. A range for the parameter options will be displayed below the box. The left number is the lower limit of the parameter. The middle is an incrementor. The right box is the upper limit of the parameter. The chosen process will be added for each parameter combination until the upper limit of all parameters is met.

4.      Adding the process. To add the process, click the “Add Process” button. This will add the process/parameter combinations to the table.

5.      Deleting Processes. Once a process is in the table, a checkbox appears next to the name. Processes can be deleted by selection with the “Delete Checked Process” button. All processes can be deleted at once using the “Delete All Processes” button.

6.    Processes can move up or down rows using the Up, Down, Decrease Stage, and Increase Stage options.
